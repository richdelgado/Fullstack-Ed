Es una plataforma que nos permite guardar repositorios de Git que podemos usar como servidores remotos y ejecturar algunos comandos de forma visual e interactiva (sin necesidad de la consola de comandos)

Nota: Git es tu repositorio local (En tu computadora) y Gitgub es en un servidor remoto (Idealmente para trabajo en equipo) y la relacion entre ambos es indispensable para ejecutar proyectos de desarrollo

Al crear un repositorio en Github se puede añadir un archivo README.me (es una buena practica) y sirve para describir el proyecto, los requerimientos y las instrucciones que debmos seguir para contribuir correctamente

  • Registrarse en Github
  • Import repository - sirve para importarun repositorio incluso de otros sistemas (solo funciona si esta online)
  • New repository - Crea un repositorio nuevo puede ser publico o privado
  • New organization - Sirve para empresas (pueden ser multiples)/li>
  • New project - es un grupo de respositorios (pueden ser dentro de una organizarion o no)
  • New gust - es para compartir un pedazo de codigo

¿Como añadir al respositorio de Github (Online) todos los cambios que tenemos listos en Git (computadora)?

  • Ir a la terminal y asegurarnos de estar en la branch y la carpeta donde esta nuestro repositorio
  • Ir a Github y dentro del repositorio ir a "Clone or Download"
  • Clone with HTTPS es copiar la URL e ir a la terminal e ingresar el comando "git remote add origin + la URL previamente copiada"
  • Para validar los cambios podemos ingresar el comando "git remote" y debemos ver origin, tambien podemos usar el comando "git remote -v" y debemos ver valores de fetch (traer) y push (enviar)
  • Finalmente ingresamos el comando "git push origin master"

Si como resultado arroja el siguiente error: "Updates were rejected because the remote contains work that you do hint: not have locally. This is usually caused by another repository pushing hint: to the same ref. You may want to first integrate the remote changes hint: (e.g., 'git pull ...') before pushing again. hint: See the 'Note about fast-forwards' in 'git push --help' for details." Esto se debe a que hay que integrar los cambios remotos antes de hacer un Push (no sabe que paso en el mundo remoto). Puedes seguir estos pasos:

  • Ir a la terminar e ingresar el comando "git pull origin + nombre de la branch" para traer los cambios al repositorio local
  • Posteriormente ingresamos el comando "git push origin + nombre de la branch" y de esta forma todo se pasa a Github
